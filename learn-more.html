ADL xAPI Launch

Learn More

xAPI Launch is an algorithm that can be implemented to launch and initialize "xAPI Launch enabled" content.

It ensures that statements which claim to be part of an experience really came from that experience. xAPI Launch differs from other launch algorithms in a few important ways.

  • It avoids placing personally indentifying information (PII) in a query string. This helps protect the learner's identity.
  • It also avoids initiating the launch session from an HTTP POST request. The body of a post request is not visible when using a URL to launch a native application.
  • In order to avoid both PII in the query string and an initial POST, it generates a one time use random token that can be exchanged for the name and LRS endpoints of the learner, but can be safely transfered as part of the URL.

Supported Uses
  • Static HTML resources
  • HTML content rendered by a server
  • Desktop applications through URL Protocol Handle
  • Mobile applications through URL Protocol Handler
  • Disconnected experiences by manually entering a code
